# Project Initiation：

> **背景说明**：数据分析师能力产品化探索（MVP 实践）<br>
> **时间**：2026年1月

## 1. 背景与动因
当前团队资源有限，数据分析师需独立承担从需求理解、方案设计、开发实现到交付验证的完整闭环工作。在此背景下，为响应“将个人核心能力产品化、规划化、规模化”的目标，亟需通过一个可落地、可复用的最小可行产品（MVP）来验证并展示该能力。

## 2. 目标
- **能力产品化**：将数据分析中的通用方法论封装为自助式工具，降低业务使用门槛。
- **流程规范化**：建立从需求定义 → 技术实现 → 部署验证 → 文档沉淀的标准路径。
- **模式可复制**：形成一套轻量级数据产品孵化模板，未来可快速其他场景。

## 3. 首个MVP选型
选择【AB实验样本量计算器】作为首个Demo。

## 4. 实施约束
- 人力：单人全栈开发（前端 + 后端 + 部署）；
- 资源：仅依赖组内一台公用 Linux 虚拟机；
- 周期：3–5 天完成端到端 MVP；
- 技术原则：极简、可运行、可演示、可维护；

## 5. 预期产出
- 一个可访问的Web工具；
- 完整代码与部署文档；
- 用户试用反馈记录；
- 可复用的“数据产品MVP实施模板”;
> 备注：本项目不追求功能完备，而重在验证“分析师自主交付数据产品”的可行性与方法论。后续将基于此模式迭代更多工具，逐步构建团队内部的数据赋能平台。

# PRD-Lite
| 模块     | 内容                                                                 |
|----------|----------------------------------------------------------------------|
| 项目名称 | AB实验样本量计算器（MVP）                                            |
| 目标     | 快速、自助式计算AB实验所需的最小样本量，支持业务方输入参数，降低实验设计门槛 |
| 用户     | 产品经理、运营、增长团队（非技术人员）                                |
| 核心功能 | 输入：显著性水平α、统计功效1-β、基线转化率、预期提升幅度<br>输出：每组所需最小样本量、总样本量 |
| 技术边界 | 单机部署、Web界面、无用户系统、无数据库（纯计算）                    |
| 成功标准 | 能跑通端到端流程；业务同事能独立使用并验证结果正确性                 |
| 时间计划 | 3–5天完成 MVP                                                        |
